# OAS v0.18.0

One public release, built from one tag: `@oas-framework/oas@0.18.0`,
`@oas-framework/pi@0.18.0`, and **OAS Desktop 0.18.0** (attached to this
GitHub Release).

## OAS Desktop

The Desktop app is the control panel for OAS deployments: roster/hierarchy,
brain/markdown/task/state views, terminal attach to existing tmux windows,
and — with a compatible CLI installed — spawning agents and harvesting
memory.

Downloads (assets on this release):

| Platform | Artifacts |
| --- | --- |
| macOS arm64 (Apple Silicon) | DMG + ZIP |
| macOS x64 (Intel) | DMG + ZIP |
| Linux x64 | AppImage + DEB |

Verify with `SHA256SUMS.txt`; GitHub build-provenance attestations are
published for all assets.

**Windows and Linux arm64 are not supported in 0.18.0.**

The 0.18.0 installers are **unsigned and not notarized**. On macOS, open via
right-click → Open (or `xattr -dr com.apple.quarantine` the app) the first
time. On Linux, `tmux` is a prerequisite.

### CLI compatibility

Desktop 0.18 requires an `oas` CLI with Desktop API v1: version
`>=0.18.0 <0.19.0`. Install or update:

```bash
npm install -g @oas-framework/oas@0.18.0
```

Without a compatible CLI, Desktop remains usable read-only (roster, files,
existing terminals); Spawn and Harvest are disabled with guidance to install
the CLI. The probe contract is documented in `docs/desktop-cli-api.md`; the
full install/migration/support guide is
[docs/desktop.md](https://github.com/OAS-Framework/oas/blob/main/docs/desktop.md).

## Breaking changes (root package)

This release removes the legacy web/TUI panel surfaces. Removal was approved
after the operator confirmed there were no other legacy-panel users.

- **`oas.web` capability removed.** Deployments that still declare it get a
  doctor-guided migration error: remove the `oas.web` entry from
  `oas-config.yaml` (`capabilities.additive`) and from `oas-lock.json` at
  that scope. `oas doctor`/`oas doctor --json` reports stale config entries,
  stale locks (`retiredLocks`), and leftover installed artifacts
  (`retiredArtifacts`) with the cleanup instruction.
- **`oas pane` removed.** The command now errors with a pointer to the
  Desktop app.
- **`@oas-framework/oas/control-pane` export removed.** No replacement —
  the Desktop app bundles its own server.

## New in the CLI

- `oas version [--json]` — the Desktop API v1 probe:
  `{"schemaVersion":1,"name":"@oas-framework/oas","version":"0.18.0","desktopApi":1}`.
- `oas spawn … --json` and `oas okf harvest --json` now emit the stable
  Desktop API v1 envelope (`{"schemaVersion":1,"ok":…}`), with exactly one
  JSON object on stdout and stable error codes. Non-JSON output is unchanged.

## Migration

1. Update the CLI everywhere: `npm install -g @oas-framework/oas@0.18.0`
   (the pi bridge `@oas-framework/pi` publishes in lockstep; restart pi
   sessions).
2. Run `oas doctor` at each workspace/repo scope and follow its guidance to
   remove stale `oas.web` config entries, locks, and installed artifacts.
3. Replace any `oas pane` / control-pane usage with the Desktop app
   (download above).
